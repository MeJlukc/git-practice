Введение в разработку Android приложений

История Android
Android – открытая ОС для мобильных телефонов, смартфонов, коммуникаторов, планшетных компьютеров, электронных книг, цифровых проигрывателей, наручных часов, смарт буков.
Android основан на ядре Linux.
Изначально ОС Android разрабатывалась компанией Android. В июле 2005г из поглощает компания Google. В ноябре 2007г Google инициирует создание бизнес-альянса Oha, в состав которого вошли HTC, Intel, Motorola, Nvidia и Samsung, которые начали дорабатывать и поддерживать Android.
С момента выхода 1ой версии Android происходит несколько важнейших обновлений политики безопасности системы. Эти обновления касаются исправления обнаруженных ошибок и добавления нового функционала в систему.

Лекция 15.09.25
Инструменты Android-разработчика

Разработка Android приложений в большинстве своем осуществляется на языке Java, который является объектно-ориентированным языком.
Программы на Java транслируются в байт-код (не компилируются). В C# же код транслируется в Java (машинный код), который обрабатывает байт-код и только после этого передает оборудованию инструкции в качестве интерпретатора. 
Достоинства такого способа заключаются в том, что байт-код в таком состоянии независим от ОС и оборудования, что позволяет выполнять Java приложения на любом устройстве для которого существует виртуальная машина.
Другой важной особенностью языка Java является гибкая система безопасности.
Благодаря тому, что исполнение приложения полностью контролируется виртуальной машиной, любые операции, которые превышают установленные полномочия приложения (например – попытка приложения получить несанкционированный доступ к данным или сетевым хранилищам) вызывают немедленное прерывание.
Для работы Android приложений в первую очередь необходим Java Development Kit (JDK) – бесплатно распространяемый комплект разработчика приложений на языке Java, который включает в себя стандартные библиотеки классов: примеры, документацию и различные утилиты, которые входят в JRE. В состав JDK не входит JRE.
Android Software Development Kit (ASDK) содержит множество инструментов и утилит для создания и тестирования мобильных приложений (например – можно установить Android Api любой версии).
Архитектура ОС Android 
Основным элементами архитектуры ОС Android является:
1.	Приложения. Android поставляется с набором основных приложений, которые включают в себя календарь, карты, браузер, менеджер контактов и т.д. Все эти приложений написаны на Java.
2.	Платформа приложений. Платформа Android представляет открытую платформу к разработке. Платформа Android представляет разработчикам возможность создавать гибкие инновационные приложения. Разработчики могут использовать аппаратные возможности устройства: получать информацию о местоположении, выполнять задачи в фоновом режиме, устанавливать оповещения и многое другое.

Разработчики имеют доступ к тем же самым Api функциям, что используется в основных приложениях.

Архитектура приложений разработана с целью упрощения повторного использования компонентов.

Любое приложение может «публиковать» свои возможности и так же любое другое приложение может использовать эти возможности (с учетом ограничения безопасности). Этот же механизм позволяет изменять стандартные компоненты на пользовательские.
3.	Библиотеки. Android включает в себя набор разнообразных библиотек, которые используются компонентами системы. Эти возможности доступны разработчикам в контексте применения Android Application Framework. Основные библиотеки:
- Медиа библиотеки. Предоставляют поддержку воспроизведения и записи многих аудио и видео форматов разрешения;
- surface manager. Управляет доступом к  подсистемам отображения 2D и 3D графических слоев;
- lip web code. Современный web движок, на котором построен Android браузер;
- SGL. Основной графический 2D движок;
- 3D библиотеки. Реализованы на основе OpenGL (либо аппаратное 3D ускорение, либо включено программно);
- freetime. Поддержка растровых и векторных шрифтов;
- SQL Lite. Механизм БД, доступный для всех приложений.

Лекция 22.09.25
Наполняемая среда Android

Android включает в себя набор основных библиотек, которые обеспечивают большинство функций, доступных в Java библиотеках.
Каждое Android приложение работает в своем собственном процессе со своим собственным экземпляром виртуальной машины, которая называется Dalvig.
Dalvig был написан так, чтобы устройство могло работать эффективно несколькими виртуальными машинами одновременно.
Dalvig проектировался специально под платформу Android. Виртуальная машина оптимизирована под низкое потребление памяти и работы на мобильном ПО.
Dalvig использует свой собственный байт код.
Android приложения переводятся компилятором в специальный машинно-независимый низкоуровневый код. И именно Dalvig интерпретирует и выполняет такую задачу при запуске на платформе Android.
Кроме того, с помощью специальной утилиты, входящей в состав Android SDK, Dalvig способен переводить байт коды Java в код собственного формата и так же запускать его в своей виртуальной среде.
Android основан на Linux v2.6 с основными системными службами: безопасностью управления памятью, процессами и моделями драйверов. Разработчики модифицировали ядро Linux, добавив поддержку аппаратных обеспечений, которые исполняются в мобильных устройствах и чаще всего недоступны на компьютерах.
Обзор интерфейсов Java
Основой для прикладного программирования Android – это набор на языке Java. В основе набора – пакеты, которые входят в состав языка Java, такие как: 
1.	Java.utu (утилиты), Java.lang, Java.io. Они есть на любой платформе, где могут быть запущены Java-приложениями. И не специфичны для Android. К ним так же примыкают расширения, которые не входят в состав языка: пакеты Javax.net, Javax.xml и др. 
2.	Но самым большим является набор интерфейсов, которые созданы специально для Android. Его основный пакеты: Пакеты Android.view и Android.widjet отвечают за графический интерфейс пользователя. Они содержат набор встроенных виджетов, таких как: кнопки и поля ввода, разметка расположения виджетов на экране. Так же с их помощью можно создавать простейшие Android приложения. 
3.	Для работы с примитивами рисования и графическими файлами предназначен пакет Android.graphics, а с помощью Android.animation можно создавать несложную анимацию. 
4.	Пакет Android.openGL представляет движок OpenGL. 
5.	Android.jestue осуществляет поддержку управления жестами на сенсорном экране, что позволяет распознавать жесты, а также создавать новые. 
6.	Большое количество интерфейсов предназначено для коммуникации. Пакет Android.net включает стеки сетевых протоколов высокого уровня, таких как http и sip, а также поддержку WiFi. 
7.	Пакет Android.webkit – популярный движок web браузера, который позволяет легко отображать web страницы в приложении. 
8.	Пакеты Android.bluetooth и Android.nfc представляют стеки протоколов связи на коротких расстояниях NFC (Near Feel Communication). 
9.	Пакет Android.telephone – доступ к телефонной функциональности (например отправка sms). 
10.	Для управления прикладными приложениями предназначен пакет Android.app.

12.	Android.location предоставляет информацию и географических координатах устройства, в т.ч. с помощью GPS датчика. 
13.	Пакет Android.media отвечает за кодирование звуковых и видео потоков для мобильных устройств. Это до сих пор вычислительно сложная задача, которая требует качественной оптимизации.

Структура Android приложений

Android приложения могут быть простыми или сложными, но структура приложения всегда будет одинакова. Есть обязательные элементы, а есть опциональные, которые буду использоваться по мере необходимости. Само же Android приложение состоит из нескольких основных компонентов: манифест приложений (файл манифеста), набор различных ресурсов приложения и исходный код самого приложения. 
Основная структура Android приложения:
Название	Описание	Необходимость
gen	Файлы, сгенерированные самой Java (здесь находится один из важных файлов приложения – R.java)	Да
AndroidManifest.xml	Файл манифеста определяет системе основную информацию о приложении (каждое приложение должно иметь свой собственный файл манифеста)	Да
Src (source)	Каталог, в котором содержит исходный код приложения	Да
assets	Произвольное собрание каталогов и файлов	Нет
Rs (resource system)	Каталог, который содержит ресурсы приложения. В данном каталоге находятся разные sup-директории (пример – values, layout, menu и т.д.)	Да

Файл манифеста несет в себе основные данные о разработке разрабатываемом приложении. У каждого приложения есть свой манифест. Редактировать данный файл можно вручную, изменяя xml код через визуальный редактор. Назначение файла: 
1.	Описывает компоненты приложения – страницы, сервисы, проводники контента и события. 
2.	Содержит список необходимых разрешений для обращения к защищенным частям api и взаимодействия с другими приложениями.
3.	Объявляет разрешения, которые сторонник приложения обязаны иметь для взаимодействия с компонентами данного приложения.
4.	Объявляет минимальный уровень api Android, который необходим для работы приложения.
5.	Перечисляет все используемые библиотеки и компоненты. 
Корневым элементом манифеста является <manifest>. Помимо данного элемента обязательными элементами являются теги. Тег <application> и <user-sdk>. Application является основным элементом манифеста и содержит множества дочерних элементов, которые определяют работу и структуру приложения. 
Порядок расположения элементов (тегов) произвольны, т.к. все значения устанавливаются через атрибуты элементов. 
Также в файле манифеста необходимо использовать следующие элементы:
1.	Xmlss android – определяет пространство имен Android.
2.	Package – определяет уникальное имя пакета приложения.
3.	Android version code – указывает на внутренний номер версии. 
4.	Android version name – указывает номер пользовательской версии.
5.	Permissions – один из важных. Объявляет разрешение, которое будет использоваться для ограничения доступа к определенным компонентам и функциональности разрабатываемого приложения. Здесь же описываются права, которые должны запросить другие приложения для получения доступа к текущему приложению.
Permission – для доступа к приложению, user.pemissions – для тех кто использует.
Приложение может так же защитить свои собственные компоненты. Может использовать любое из системных разрешений, определенных Android или объявленных другими приложениями, а также можно определять свои собственные разрешения
6.	Users.permissions – приложения запрашивают разрешения, которые приложению должны быть представлены системой Android (пользователем) для его нормального функционирования. Как правило данные разрешения предоставляются во время установки приложения., но никак не во время его работы. Наиболее распространены следующие виды разрешений:
6.1.	Интернет – доступ к сети интернет.
6.2.	ReadContacts – чтение (но не запись) данных из адресной книги пользователя. 
6.3.	WriteContacts – запись (но не чтение) данных в адресную книгу пользователя.
6.4.	OresSMS – обработка входящих SMS. 
6.5.	AccessFileLocation – точное определение местонахождения при помощи GPS.

Экран для мобильного устройства, размеров экрана, языков интерфейса.
Так же элементы интерфейса можно создавать и программно, когда это необходимо. Каждый файл разметки должен содержать только один корневой элемент, который должен быть объектом view или view group. Внутри корневого объекта можно добавить дополнительные элементы разметки или дочерние элементы интерфейса, чтобы постепенно формировать иерархию элементов, которая определяет создаваемая разметка.
Существует несколько стандартных типов разметок:
1.	Frame Layout – самый простой тип разметки. Обычно это пустое пространство на экране, которое можно заполнить только дочерним объектом view или view group. Все дочерние элементы этого типа разметки прикрепляются к верхнему левому углу экрана.
